# Cogochi Pokemon-Grade Autoresearch Plan

Last updated: 2026-03-13

## Objective

Raise Cogochi from a connected prototype into a `Pokemon-grade alpha`:

- world-first
- encounter-driven
- squad-attached
- visibly evolving
- chart-native

The target is not content parity with Pokemon.
The target is parity in:

- loop readability
- world and battle rhythm
- player attachment
- clarity of progression

## Product Outcome

The branch is on target only if a new player can say:

1. `I knew where to go.`
2. `I walked my squad there.`
3. `I hit one clear clash objective.`
4. `I made a mutation verdict.`
5. `My creature visibly changed or clearly needs repair.`
6. `I knew what to do next without menu confusion.`

## Phase Map

### Phase 1. World-First Loop

Goal:

- make `/field` the main game surface

Deliverables:

- compact route structure
- camp orientation
- one support node
- one spar gate
- one proof gate
- verdict-aware return

Exit criteria:

- a full loop can start and end on `/field`

### Phase 2. Encounter Legibility

Goal:

- make clashes read like game encounters instead of reports

Deliverables:

- one primary objective per clash
- stronger world-space danger and success cues
- command deck readable in two seconds
- less side-panel dependence

Exit criteria:

- a viewer can tell what the fight is about without reading long text

### Phase 3. Evolution Visibility

Goal:

- make growth undeniable

Deliverables:

- stage growth silhouettes
- mutation accessory sets
- proven-frame marks
- failure scars and care states

Exit criteria:

- one proof run visibly changes either body or history on return

### Phase 4. Return-Pressure Surfaces

Goal:

- make the post-battle loop as strong as the battle

Deliverables:

- `/journal`
- camp journal board
- latest mutation summary
- next care action prompt

Exit criteria:

- after returning, the player sees one compelling next action immediately

### Phase 5. Alpha Polish Gates

Goal:

- tighten the whole loop into a five-minute alpha

Deliverables:

- first five minute onboarding
- route pacing pass
- text reduction pass
- HUD simplification pass
- screenshot and browser evidence

Exit criteria:

- the first loop is readable, short, and replayable

## Evaluation Rules

### Hard Gates

1. Field remains chart-native
2. Battle remains deterministic
3. Mutation verdict loop remains explicit
4. World-first loop remains intact
5. Docs and context gates pass

### Weighted Priorities

1. world-first session loop
2. encounter legibility
3. visible growth and return pressure
4. short replayable session rhythm
5. explanation and trust clarity
6. polish

## Evidence Expectations

Every phase should leave:

- screenshots
- text state outputs
- one short watch-log summary
- one updated bottleneck ranking

## Stop Conditions

Do not declare the alpha complete if any of these remain true:

- the player still starts by reading panels
- the next goal is not obvious
- the first clash takes too long to reach
- the return loop does not surface a next action
- the creature does not visibly change over repeated proof runs
